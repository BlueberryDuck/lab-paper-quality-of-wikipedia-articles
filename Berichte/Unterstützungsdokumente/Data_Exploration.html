Datenexploration für den Datensatz Wikipedia Promotional Articles¶

In dieser Analyse wird der Datensatz Wikipedia Promotional Articles untersucht. Der Datensatz enthält Informationen über Wikipedia-Artikel, die als "promotional" oder "good" kategorisiert sind.

Ziel der Exploration:¶

  • Übersicht über die Daten gewinnen.
  • Relevante Muster identifizieren.
  • Herausforderungen für das ML-Projekt identifizieren.

Beschreibung des Datensatzes und seiner Labels¶

Der Datensatz besteht aus Artikeln, die von Nutzern entweder mit einem "promotional tone" oder als "good articles" markiert wurden. Die entsprechenden Daten sind in zwei separaten Dateien enthalten:

  • promotional.csv: Enthält Artikel, die als werbend oder einseitig wahrgenommen wurden.
  • good.csv: Enthält Artikel, die als qualitativ hochwertig und neutral eingestuft wurden.

Labels im Datensatz¶

Jeder Artikel in der Datei promotional.csv kann mehrere der folgenden Labels besitzen, die Zitate aus den entsprechenden Wikipedia-Tags darstellen:

  1. advert
    "This article contains content that is written like an advertisement."
    Dieses Label kennzeichnet Artikel, die sich in ihrem Schreibstil wie Werbetexte lesen.

  2. coi
    "A major contributor to this article appears to have a close connection with its subject."
    Dieses Label wird vergeben, wenn ein Hauptautor des Artikels eine enge Verbindung zum Thema hat, was die Neutralität beeinträchtigen könnte.

  3. fanpov
    "This article may be written from a fan's point of view, rather than a neutral point of view."
    Artikel mit diesem Label könnten aus der Perspektive eines Fans geschrieben sein, wodurch eine objektive Darstellung fehlt.

  4. pr
    "This article reads like a press release or a news article or is largely based on routine coverage or sensationalism."
    Dieses Label weist darauf hin, dass der Artikel wie eine Pressemitteilung oder ein reißerischer Nachrichtenbericht geschrieben ist.

  5. resume
    "This biographical article is written like a résumé."
    Artikel mit diesem Label haben einen biografischen Charakter, der eher an einen Lebenslauf erinnert, anstatt eine objektive Darstellung zu bieten.

Erkenntnisse aus der Datenanalyse¶

  • Promotional Articles: Dieser Datensatz enthält insgesamt 23.837 Artikel mit den sieben Spalten:
    text, advert, coi, fanpov, pr, resume und url. Die Spaltenwerte sind vollständig, es gibt keine fehlenden Einträge. Auch enthalten die Labels advert, coi, fanpov, pr, resume nur die Werte 0 oder 1 womit ausgedrückt werden soll, ob das Label für den Text zutrifft. Damit sind nur gültige Werte enthalten.

  • Good Articles: Dieser Datensatz umfasst 30.279 Artikel mit den beiden Spalten:
    text und url. Auch in diesem Datensatz gibt es keine fehlenden Werte.

labels = ['advert', 'coi', 'fanpov', 'pr', 'resume']

exclusive_label_counts = {}
for label in labels:
    # Prüfen, ob nur dieses Label den Wert 1 hat und alle anderen Labels den Wert 0
    exclusive_count = promotional_articles[
        (promotional_articles[label] == 1) & 
        promotional_articles[[col for col in labels if col != label]].sum(axis=1).eq(0)
    ].shape[0]
    exclusive_label_counts[label] = exclusive_count

# Gesamtanzahl der Artikel mit Einzel-Labels berechnen
total_exclusive = sum(exclusive_label_counts.values())

import matplotlib.pyplot as plt
import pandas as pd

# Labels
labels = ['advert', 'coi', 'fanpov', 'pr', 'resume']

# Berechnung der Anzahlen für Einzellabel- und Mehrfachlabel-Artikel pro Label
ratios = {}
for label in labels:
    # Artikel mit nur diesem Label
    single_label = promotional_articles[
        (promotional_articles[label] == 1) &
        promotional_articles[[col for col in labels if col != label]].sum(axis=1).eq(0)
    ].shape[0]
    
    # Artikel mit diesem Label, aber auch anderen Labels
    multi_label = promotional_articles[
        (promotional_articles[label] == 1) &
        promotional_articles[[col for col in labels if col != label]].sum(axis=1).gt(0)
    ].shape[0]
    
    # Sicherstellen, dass Single- und Multi-Label korrekt summieren
    total = single_label + multi_label
    ratios[label] = {'Einzellabel': single_label, 'Mehrfachlabel': multi_label, 'Total': total}

# Daten in einen DataFrame für bessere Übersicht
ratios_df = pd.DataFrame(ratios).T[['Einzellabel', 'Mehrfachlabel']]

# Berechnung der Prozentzahlen
ratios_percent = ratios_df.div(ratios_df.sum(axis=1), axis=0) * 100

# Gestapeltes Balkendiagramm
ax = ratios_df.plot(kind='bar', stacked=True, color=['blue', 'yellow'], figsize=(10, 6))

# Prozentzahlen hinzufügen neben dem Balken für kleine Werte
for i, label in enumerate(ratios_df.index):
    total = ratios_df.loc[label].sum()
    ein_percent = ratios_percent.loc[label, 'Einzellabel']
    multi_percent = ratios_percent.loc[label, 'Mehrfachlabel']

    # Einzellabel: Position und Schriftfarbe
    ein_y = ratios_df.loc[label, 'Einzellabel'] / 2
    if ein_percent < 1:  # Prozentzahl neben den Balken platzieren
        ax.text(i + 0.2, ratios_df.loc[label, 'Einzellabel'], 
                f"{ein_percent:.1f}%", color='black', ha='left', va='center', fontsize=10)
    else:
        ax.text(i, ein_y,
                f"{ein_percent:.1f}%", color='white', ha='center', va='center', fontsize=10)

    # Mehrfachlabel: Position und Schriftfarbe
    multi_y = ratios_df.loc[label, 'Einzellabel'] + ratios_df.loc[label, 'Mehrfachlabel'] / 2
    if multi_percent < 1:  # Prozentzahl neben den Balken platzieren
        ax.text(i + 0.1, ratios_df.loc[label, 'Einzellabel'] + ratios_df.loc[label, 'Mehrfachlabel'],  # Neben Balken
                f"{multi_percent:.1f}%", color='black', ha='left', va='center', fontsize=10)

import pandas as pd
from itertools import combinations


# Labels außer coi
other_labels = ['advert', 'fanpov', 'pr', 'resume']

# Funktion, um die Anzahl von Artikeln für jede Kombination von Labels zu berechnen
def count_combinations(label, other_labels, df):
    results = []
    # Einzeln
    for single in other_labels:
        count = df[(df[label] == 1) & (df[single] == 1) & df[[col for col in other_labels if col != single]].sum(axis=1).eq(0)].shape[0]
        results.append({'Kombination': f'{single}', 'Anzahl': count})
    # Paarweise
    for pair in combinations(other_labels, 2):
        count = df[(df[label] == 1) & df[list(pair)].sum(axis=1).eq(2) & df[[col for col in other_labels if col not in pair]].sum(axis=1).eq(0)].shape[0]
        results.append({'Kombination': ' & '.join(pair), 'Anzahl': count})
    # Dreifach
    for triple in combinations(other_labels, 3):
        count = df[(df[label] == 1) & df[list(triple)].sum(axis=1).eq(3) & df[[col for col in other_labels if col not in triple]].sum(axis=1).eq(0)].shape[0]
        results.append({'Kombination': ' & '.join(triple), 'Anzahl': count})
    # Alle vier
    count_all = df[(df[label] == 1) & df[other_labels].sum(axis=1).eq(4)].shape[0]
    results.append({'Kombination': ' & '.join(other_labels), 'Anzahl': count_all})
    return pd.DataFrame(results)

Ergebnis¶

Es gibt eine sehr gute Trennung zwischen Artikeln mit den Label Good bzw. Promotional.Dadurch werden sich leicht gute einfache Modelle bauen lassen. Es fehlen keine Einträge und es muss somit kein Data Cleaning vorgenommen werden.

Herausforderungen bestehen bei der Klassifikation der Promotional Artikeln. Das advert Sublabel ist so dominant, dass im Zweifel dieses immer angenommen wird. coi hingegen hat nur drei eigene Labels wodurch es quasi verschwindet bei der Vielzahl von Überschneidungen mit den anderen Sublabels. Es wird also zu erwarten sein, dass coi nur sehr schlecht aufzufinden ist.
